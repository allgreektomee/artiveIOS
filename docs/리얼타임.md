# 리얼타임 (WebSocket) 문서

네이티브 WebSocket(`/ws/realtime?token=...`) 수신부터 UI·세션 처리까지, 현재 구현을 **파일·타입 단위**로 정리한 문서이다.

---

## 1. 서버 (Java, `artiveB/me`)

| 파일 | 역할 |
|------|------|
| `com.artivefor.me.realtime.RealtimePushMessage` | 공통 JSON: `type`, `message`, `reason`, `timestamp`. `type` 상수: `CONNECTION_TEST`, `SESSION_SUPERSEDED`. |
| `RealtimeHeartbeatScheduler` | 주기적으로 `CONNECTION_TEST` 브로드캐스트. |
| `LoginSessionService` | 소켓 등록/해제, 중복 로그인 시 이전 세션에 `SESSION_SUPERSEDED` 전송 후 연결 종료. |
| `RealtimeWebSocketHandler` | 연결 수립·종료 시 `LoginSessionService` 연동. |
| `JwtHandshakeInterceptor` | 핸드셰이크 시 JWT 속성(이메일, `testSessionId` 등) 주입. |

클라이언트는 **`type` 문자열**으로 메시지 종류를 구분한다.

---

## 2. iOS 수신 흐름 (한 줄 요약)

`URLSessionWebSocketTask.receive` → `RealtimeSocketService.handlePayload(_ jsonString:)` → `RealtimeSocketEnvelope.parse(jsonString:)` → `RealtimeSocketPayloadRouter.route(_:context:)` → `type`에 대응하는 **핸들러** `handle(_:context:)`.

---

## 3. iOS 코드 구조

### 3.1 진입점: `RealtimeSocketService.swift`

- `connect(accessToken:)` / `disconnect()`: 웹소켓 태스크 생명주기.
- `receiveLoop()`: `.string(jsonString)` / `.data`→UTF-8 문자열 후 동일하게 `handlePayload(jsonString)` 호출.
- `handlePayload(_ jsonString:)`:
  - `RealtimeSocketEnvelope.parse(jsonString:)` 실패 시 로그 후 return.
  - 성공 시 `RealtimeSocketPayloadContext(appendLog:notifySessionSuperseded:)` 생성.
  - `RealtimeSocketPayloadRouter.route(envelope, context:)` 호출.
- `onSessionSuperseded: (() -> Void)?`: 프로토콜에 선언. **세션 축출** 전용 콜백(아래 5절).

### 3.2 봉투: `RealtimeSocket/RealtimeSocketEnvelope.swift`

- **역할**: 서버가 준 UTF-8 **JSON 문자열**을 최소 단위로 파싱.
- **필드**
  - `type: String` — JSON의 `"type"`.
  - `dictionary: [String: Any]` — 객체 전체(핸들러가 필드별로 해석).
  - `rawJSONString: String` — 원문 보관(알 수 없는 타입 로그 등).
- **팩토리**: `static func parse(jsonString: String) -> RealtimeSocketEnvelope?`
  - JSON이 아니거나 `"type"`이 없으면 `nil`.

### 3.3 라우터: `RealtimeSocket/RealtimeSocketPayloadRouter.swift`

- **`RealtimeSocketPayloadContext`**
  - `appendLog: (String) -> Void` — 실시간 로그 유즈케이스로 한 줄 추가.
  - `notifySessionSuperseded: () -> Void` — 세션 축출 시 루트에서 후속 처리(네비·스토리지 정리 연동).
  - 여러 `type`이 **같은 context 타입**을 공유한다. 각 핸들러는 필요한 클로저만 호출한다.

- **`RealtimeSocketPayloadHandling`**
  - `var type: String { get }` — 서버 JSON의 `type`과 **동일한 라우팅 키**.
  - `func handle(_ envelope: RealtimeSocketEnvelope, context: RealtimeSocketPayloadContext)`

- **`RealtimeSocketPayloadRouter`**
  - `private static let handlers: [String: any RealtimeSocketPayloadHandling]`  
    앱 로드 시 한 번: 등록된 핸들러 목록 → `Dictionary( type → 핸들러 )`.
  - `route(_ envelope:context:)`: `handlers[envelope.type]`이 있으면 `handle`, 없으면 `handleUnknown` (로그).

### 3.4 핸들러: `RealtimeSocket/Handlers/*.swift`

| 파일 | `type` | 하는 일 |
|------|--------|---------|
| `ConnectionTestPayloadHandler.swift` | `CONNECTION_TEST` | `timestamp`, `message` 로깅. |
| `SessionSupersededPayloadHandler.swift` | `SESSION_SUPERSEDED` | 로깅 후 `DispatchQueue.main.async { context.notifySessionSuperseded() }`. |

**새 실시간 기능 추가 시**

1. `Handlers/`에 `…PayloadHandler.swift` 추가, `RealtimeSocketPayloadHandling` 준수, `let type = "서버와_동일한_문자열"`.
2. `RealtimeSocketPayloadRouter`의 `handlers` 배열에 인스턴스 한 줄 추가.
3. 서버에서 해당 `type`의 JSON 전송 (`RealtimePushMessage` 등).

### 3.5 로그 UI·유즈케이스

| 파일 | 역할 |
|------|------|
| `05Use/UseRealtimeSocketLog.swift` | `@Published` 로그 라인 버퍼, `appendLog` 등. |
| `06Page/RealtimeSocketLogView.swift` | `EnvironmentObject`로 로그 표시. |
| `ServiceModule.swift` | **`UseRealtimeSocketLog`를 `RealtimeSocketService`보다 먼저** 등록(주입 순서). |

`RealtimeSocketService`는 `emitLogLine`에서 `useRealtimeLog.appendLog` 호출. **소켓 로그용 `NotificationCenter`는 사용하지 않는다.**

## 4. 세션 축출 (`SESSION_SUPERSEDED`) — NotificationCenter 제거

이전에는 `Notification.Name.artiveSessionSuperseded`를 쓰는 경로가 있었으나, **콜백 + `NavigationManager` 상태**로 통일하였다.

- **삭제**: `artive/common/ArtiveNotifications.swift` (세션 superseded 전용 노티 정의만 있던 파일).
- **`SessionSupersededPayloadHandler`**: `NotificationCenter.post` 없음. `context.notifySessionSuperseded()`만 호출.
- **`RealtimeSocketService`**: `notifySessionSuperseded` 클로저가 `onSessionSuperseded?()` 호출.
- **`RootView.swift`**
  - `.onAppear`: `RealtimeSocketServiceProtocol.onSessionSuperseded = { navManager.signalSessionSuperseded() }`
  - `.onChange(of: navManager.sessionSupersededEpoch)`: `disconnect()`, `ServiceStorage` `clearAll()`, 강제 로그아웃 알림.
- **`NavigationManager.swift`**: `sessionSupersededEpoch`, `signalSessionSuperseded()`.

**주의**: `onSessionSuperseded`는 `RootView.onAppear` 이후에 설정된다. 소켓이 그보다 먼저 메시지를 받는 극초기 레이스는 앱 구조상 드물며, 필요 시 앱 진입점에서 동일 할당을 앞당길 수 있다.

## 5. 네이밍 정리 (요약)

| 개념 | 이름 |
|------|------|
| 서버 JSON 문자열 인자 | `jsonString` (`parse`, `handlePayload`) |
| 파싱 결과에 넣는 원문 | `rawJSONString` |
| 핸들러·서버와 맞추는 분기 키 | `type` (`RealtimeSocketPayloadHandling.type`, `RealtimeSocketEnvelope.type`) |
